How does Redux Work?

Redux Flow

Create provider in app root

Store

  • Holds application state
  • Allows access to state via getState()
  • Allows state to be updated via dispatch(action)

How do you create a store

import { createStore } from 'redux'
import todoApp from './reducers'
let store = createStore(todoApp)

In React Component

Use connect to map the stores state and dispatch to the props of a component

const mapStateToProps = state => {
  return {todo : state.todos[0]}
}

const mapDispatchToProps = dispatch => {
  return {
    destroyTodo : () => dispatch({type : 'DESTROY_TODO'})
  }
}
export default connect(
    mapStateToProps,
    mapDispatchToProps
)(TodoItem)

Dispatch Actions

To change data you dispatch an action

const mapDispatchToProps => dispatch => {
  return {
    getLinks: (topicName) => dispatch(getLinks(topicName)),
    startAdd: (topicName) => dispatch(startAdd(topicName)),
  };
}

Selectors

  • Compute derived data, allowing Redux to store the minimal possible state
  • Are composable and they can be used as input to other selectors
import { createSelector } from 'reselect'

const getVisibilityFilter = (state, props) =>
  state.todoLists[props.listId].visibilityFilter

const getTodos = (state, props) =>
  state.todoLists[props.listId].todos

const getVisibleTodos = createSelector(
  [ getVisibilityFilter, getTodos ],
  (visibilityFilter, todos) => {
    switch (visibilityFilter) {
      case 'SHOW_COMPLETED':
        return todos.filter(todo => todo.completed)
      case 'SHOW_ACTIVE':
        return todos.filter(todo => !todo.completed)
      default:
        return todos
    }
  }
)

export default getVisibleTodos

Actions

  • Action is just a plain js object
  • Every change is an Action
export function requestLinks(topicName) {
  return {
    type: REQUEST_LINKS,
    topicName,
  };
}

Actions

  • Create send data from your application to your store
  • Only source of info for the store
  • Send to store using store.dispatch()
  • Must contain type

Reducers

  • Tie state and actions together with Reducers
  • Function that takes state and action as argument
  • Returns next state
function linkListContainerReducer(state, action) {
  switch (action.type) {
    case REQUEST_LINKS_SUCCEEDED:
      return state.set('links', action.links);
    case ADD_LINK_SUCCESS:
      return addLink(state, action.link);
    default:
      return state;
  }
}

Reducers

  • Specify how the application state changes in response to an action
  • Based on array.reduce()
  • Keep data separate from the UI state
  • Pure function
  • (previousState, action) => newState
  • Think of app's state as a database
  • Keep entities in objects with an ID
  • Use IDs to reference it from other entities, or lists

Things you should never do in a Reducer

  • Mutate its arguments
  • Perform side effects like API calls and routing transitions
  • Call non-pure functions, e.g. Date.now() or Math.random()

How to use reducers

  • import the actions
  • Initialize state
  • Use switch statement to catch action
  • Change state use object.assign without mutating it
  • Use default to pass the previous state
